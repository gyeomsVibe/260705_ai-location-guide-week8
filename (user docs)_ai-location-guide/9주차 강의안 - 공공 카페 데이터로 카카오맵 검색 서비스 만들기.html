공공데이터포털에서 받은 카페 데이터를 활용해, 내 위치 주변 카페를 검색하고 카카오맵에 표시하는 웹앱을 Antigravity 2.0 에이전트에게 지시해서 완성한다.
전제: 8-1주차처럼 Render 공개 URL과 카카오맵 기본 구현은 이미 완료되어 있다고 보고 시작합니다. 오늘은 “지도 위에 카페 데이터를 검색해서 보여주는 기능”을 붙이는 날입니다. 🚀
오늘도 참여자는 소스 코드를 직접 작성하지 않습니다. 설치, 수정, 실행, 오류 해결은 모두 Antigravity 2.0 에이전트에게 지시합니다.

실습 1단계: 기존 프로젝트 상태 확인하기

먼저 8-1주차에서 만든 카카오맵 프로젝트를 Antigravity 2.0에서 엽니다.

  1. Antigravity 2.0을 실행합니다.
  2. Open Folder 또는 프로젝트 열기로 8-1주차 프로젝트 폴더를 엽니다.
  3. AI 에이전트 채팅창을 엽니다.
  4. 아래 프롬프트를 그대로 붙여넣습니다.
이 프로젝트는 8-1주차에서 만든 카카오맵 웹앱입니다. 제가 소스 코드를 직접 수정하지 않도록, 현재 프로젝트 상태를 먼저 점검해 주세요. 확인할 것: 1. 카카오맵이 정상적으로 로딩되는지 2. Render 배포용 설정이 되어 있는지 3. GitHub 연결 상태가 정상인지 4. 서버 실행 방법과 필요한 설치 패키지가 무엇인지 5. 앞으로 공공데이터포털 카페 데이터를 연결하기 위해 어떤 구조가 적절한지 필요한 설치나 실행은 에이전트가 진행해 주세요. 소스 코드 설명은 길게 하지 말고, 제가 해야 할 클릭/입력 행동만 알려주세요.
✅ 성공 기준: 에이전트가 프로젝트 실행 방법을 확인하고, 로컬 또는 미리보기 화면에서 기존 카카오맵이 보이면 성공입니다.

실습 2단계: 공공 카페 데이터 연결 준비시키기

카페 API 인증키를 에이전트에게 전달합니다. 단, 인증키를 프론트 화면에 직접 노출하지 않도록 요청합니다.

인증키를 채팅창에 붙여넣는 것은 실습 환경에서는 가능하지만, 화면 캡처나 공유 자료에는 절대 노출하지 않습니다. 가능하면 에이전트에게 .env 또는 Render 환경변수로 저장하게 지시합니다.
공공데이터포털에서 받은 다음 API를 이 프로젝트에 연결해 주세요. 서비스 이름: 소상공인시장진흥공단_상가(상권)정보_API 목표: 전국 상가 데이터 중에서 카페/커피/음료 업종만 검색해서 카카오맵에 표시하고 싶습니다. 중요 조건: 1. 저는 코드를 직접 수정하지 않습니다. 필요한 파일 수정은 에이전트가 해주세요. 2. 공공데이터 인증키는 화면에 노출하지 말고 안전한 방식으로 보관해 주세요. 3. 로컬 실행용 환경변수와 Render 배포용 환경변수 설정 방법을 각각 알려주세요. 4. 필요한 패키지가 있으면 에이전트가 설치해 주세요. 5. 먼저 카페 데이터 5개만 조회하는 테스트 기능을 만들어 주세요. 6. 테스트가 성공하면 결과에 상호명, 업종명, 주소, 위도, 경도가 있는지 확인해 주세요. 제가 인증키를 입력해야 하면, 어떤 키를 어디에 붙여넣으면 되는지만 알려주세요.

참여자가 에이전트에게 전달할 정보

메모장에 정리해 둘 내용
  • 공공데이터 서비스 이름: 소상공인시장진흥공단_상가(상권)정보_API
  • 인증키: 공공데이터포털에서 복사한 인증키
  • 원하는 데이터: 카페, 커피, 음료 업종
  • 지도 표시 필드: 상호명, 주소, 위도, 경도
✅ 성공 기준: 에이전트가 카페 데이터 조회 테스트를 하고, 5개 내외의 카페 후보 데이터를 보여주면 성공입니다.

실습 3단계: 첫 화면을 “내 위치 기준 1km”로 만들기

사용자가 앱에 처음 들어오면 현재 위치를 가져오고, 지도 중심을 내 위치로 이동시킨 뒤 1km 원을 표시합니다.

첫 화면 기본 상태를 다음처럼 만들어 주세요. 기본 화면 요구사항: 1. 사용자가 웹앱에 접속하면 브라우저 위치 권한을 요청해 주세요. 2. 위치 권한을 허용하면 내 위치를 지도 중심으로 설정해 주세요. 3. 내 위치 마커를 표시해 주세요. 4. 기본 검색 반경은 1km로 설정해 주세요. 5. 지도 위에 내 위치 기준 1km 원을 표시해 주세요. 6. 1km 안에 있는 카페만 기본으로 표시해 주세요. 7. 위치 권한을 거부하면 기본 위치를 하나 정해서 안내 문구를 보여주세요. 8. HTTPS 공개 URL에서는 위치 기능이 정상 작동해야 합니다. 저는 코드를 직접 보지 않을 것이므로, 에이전트가 구현하고 실행해서 확인해 주세요.
초보자가 많이 막히는 부분: 위치 권한 팝업에서 “차단”을 누르면 현재 위치를 못 가져옵니다. 이 경우 브라우저 주소창 왼쪽의 권한 설정에서 위치를 다시 허용해야 합니다.
✅ 성공 기준: 첫 화면에 내 위치 마커, 1km 원, 주변 카페 마커가 함께 보이면 성공입니다.
4

버튼처럼 보이지만 딱딱한 버튼이 아니라, 모바일 앱에서 자주 보는 라벨/칩 스타일로 만듭니다.

검색 반경을 바꿀 수 있는 라벨식 버튼을 추가해 주세요. 라벨 버튼: - 1km - 3km - 5km - 전국 동작 요구사항: 1. 처음 기본 선택은 1km입니다. 2. 1km, 3km, 5km를 누르면 지도 원의 크기와 검색 결과가 함께 바뀌어야 합니다. 3. 선택된 라벨은 색상으로 강조해 주세요. 4. 전국을 누르면 반경 원은 숨기거나 “전국 검색 모드”로 표시해 주세요. 5. 전국 검색은 데이터가 너무 많을 수 있으니 한 번에 너무 많이 표시하지 말고, 검색어 입력 후 결과 일부만 보여주게 해주세요. 6. 지도 마커와 왼쪽/아래쪽 결과 목록이 함께 갱신되게 해주세요. 7. 모바일에서도 손가락으로 누르기 쉽게 큼직한 칩 UI로 만들어 주세요.
✅ 성공 기준: 1km, 3km, 5km, 전국 라벨을 누를 때마다 선택 표시와 검색 결과가 바뀌면 성공입니다.
5

실습 5단계: 카페 검색창 만들기

상호명, 주소, 업종명에 검색어가 들어가면 결과가 나오도록 요청합니다.

카페 검색 기능을 추가해 주세요. 검색창 요구사항: 1. 검색어 입력창과 검색 버튼을 만들어 주세요. 2. 사용자가 “스타벅스”, “커피”, “라떼”, “강남” 같은 단어를 입력하면 관련 카페를 찾아 주세요. 3. 현재 선택된 반경이 1km/3km/5km이면 내 위치 기준 해당 반경 안에서만 검색해 주세요. 4. “전국”이 선택되어 있으면 전국 데이터에서 검색하되, 결과가 너무 많으면 상위 일부만 보여주세요. 5. 검색 결과 목록에는 상호명, 주소, 업종명, 내 위치로부터의 거리를 표시해 주세요. 6. 검색 결과를 클릭하면 해당 카페 위치로 지도가 이동하고 마커가 강조되게 해주세요. 7. 검색 결과가 없으면 “검색 결과가 없습니다. 반경을 넓혀보세요.”라고 안내해 주세요.
✅ 성공 기준: 검색어를 입력하고 검색 버튼을 누르면 지도 마커와 목록이 함께 바뀌면 성공입니다.
6

실습 6단계: 화면을 초보자용으로 정리하기

기능이 작동하면 마지막에는 참여자가 봐도 이해되는 화면으로 정리합니다.

전체 화면을 초보자도 이해하기 쉬운 카페 지도 서비스처럼 정리해 주세요. 디자인 요구사항: 1. 제목은 “내 주변 카페 찾기”로 해주세요. 2. 상단에는 현재 검색 반경과 검색 결과 개수를 보여주세요. 3. 검색창, 반경 라벨, 결과 목록, 지도가 한눈에 보이게 배치해 주세요. 4. 모바일에서는 검색창과 라벨 버튼이 위에, 지도와 목록이 아래로 자연스럽게 보이게 해주세요. 5. 카페 마커는 일반 마커와 구분되게 커피 느낌 아이콘 또는 라벨을 사용해 주세요. 6. 로딩 중에는 “카페 데이터를 불러오는 중입니다”라고 보여주세요. 7. 오류가 나면 사용자가 이해할 수 있는 쉬운 문장으로 안내해 주세요. 8. 전체 색감은 8-1주차 카카오맵 교안과 비슷한 느낌으로 정리해 주세요.
✅ 성공 기준: 기능만 있는 화면이 아니라, 실제 서비스처럼 보이면 성공입니다. 🎉
7

실습 7단계: GitHub 반영과 Render 재배포

완성된 내용을 공개 URL에 반영합니다. Git 명령어는 직접 입력하지 않습니다.

지금까지 수정한 내용을 GitHub에 반영하고 Render 공개 URL에 재배포되도록 처리해 주세요. 요구사항: 1. 변경된 파일을 확인해 주세요. 2. 적절한 커밋 메시지로 Git 커밋을 만들어 주세요. 3. GitHub 원격 저장소에 푸시해 주세요. 4. Render에서 자동 배포가 시작되는지 확인해 주세요. 5. 배포가 끝나면 공개 URL에서 카페 검색 기능이 작동하는지 점검해 주세요. 6. 문제가 있으면 원인을 찾고 수정해 주세요. 저는 명령어를 직접 입력하지 않을 것이므로 필요한 작업은 에이전트가 진행해 주세요.
✅ 성공 기준: 휴대폰에서 Render 공개 URL을 열었을 때 내 주변 카페 검색 화면이 보이면 성공입니다.

🧯 자주 막히는 문제와 해결법

문제 1. 내 위치를 못 가져와요브라우저 위치 권한이 차단되었을 수 있습니다.

에이전트 지시: “위치 권한 거부 상황에서 안내 문구와 기본 위치 fallback을 추가해 주세요.”
문제 2. 카페 데이터가 안 나와요공공데이터 인증키, API 주소, 요청 파라미터가 틀렸을 수 있습니다.

에이전트 지시: “공공데이터 API 호출 로그를 확인하고 인증키와 요청 파라미터를 점검해 주세요.”
문제 3. 지도는 보이는데 마커가 없어요위도/경도 필드명이 다르거나 좌표 데이터가 비어 있을 수 있습니다.

에이전트 지시: “카페 데이터의 위도, 경도 필드명을 확인하고 마커 생성 로직을 수정해 주세요.”
문제 4. 전국 버튼을 누르면 느려요전국 데이터가 너무 많기 때문입니다.

에이전트 지시: “전국 검색은 검색어가 있을 때만 실행하고, 결과 개수를 제한해 주세요.”
문제 5. Render에서는 위치 기능이 이상해요브라우저 위치 기능은 보안 정책상 HTTPS에서 안정적으로 작동합니다.

에이전트 지시: “Render 공개 URL의 HTTPS 환경에서 위치 권한과 geolocation 동작을 점검해 주세요.”
문제 6. 반경 버튼이 눌려도 결과가 같아요버튼 UI와 실제 검색 반경 값이 연결되지 않았을 수 있습니다.

에이전트 지시: “반경 라벨 클릭 이벤트와 검색 함수가 연결되어 있는지 확인하고 수정해 주세요.”

🧑‍🏫 강사용 진행 멘트

“오늘도 우리는 코딩을 직접 하지 않습니다. 대신 AI 에이전트에게 일을 맡기는 법을 배웁니다.”
“카페 데이터는 정부 DB에서 가져오고, 지도 표시는 카카오맵이 해줍니다. 우리는 둘을 연결하라고 AI에게 지시하는 겁니다.”
“1km, 3km, 5km 버튼은 그냥 예쁜 버튼이 아닙니다. 데이터 검색 범위를 바꾸는 스위치입니다.”
“전국 버튼은 욕심 버튼입니다. 누르면 데이터가 많아질 수 있으니 검색어와 함께 쓰는 방식으로 만들겠습니다.”
“오류가 나도 괜찮습니다. 오류 메시지를 복사해서 에이전트에게 보여주는 것도 바이브 코딩의 중요한 능력입니다.”

📣 참여자에게 꼭 강조할 금지사항

  • 소스 코드를 직접 열어서 고치지 않습니다.
  • 인증키를 강의 화면, 캡처 화면, GitHub 공개 저장소에 노출하지 않습니다.
  • 카카오 Admin 키를 사용하지 않습니다. 카카오맵은 JavaScript 키를 사용합니다.
  • 공공데이터 API가 안 된다고 바로 포기하지 않습니다. 키, 승인 상태, 요청 주소, 파라미터를 에이전트에게 점검시킵니다.
  • 전국 데이터를 한 번에 모두 지도에 찍으려고 하지 않습니다. 느려질 수 있습니다.
