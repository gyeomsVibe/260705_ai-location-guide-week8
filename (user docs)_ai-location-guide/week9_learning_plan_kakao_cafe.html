공공 카페 데이터로
카카오맵 검색 서비스 만들기

9주차 교안을 “한 단계씩 실행하고, 각 단계의 성공 증거를 확인한 뒤 다음 단계로 이동”하도록 재구성한 순차형 학습계획서

1. 전수분석 요약

1개최종 목표: 내 위치 주변 카페 검색 웹앱
7단계상태 확인 → API → 위치 → 반경 → 검색 → UI → 배포
3시간교안 기준 수업 시간
6개주요 오류 분기

이 수업의 핵심은 코드를 직접 작성하는 것이 아니라, 기존 카카오맵 프로젝트에 공공데이터 API를 연결하도록 Antigravity 2.0 에이전트에게 기능 단위로 지시하고 결과를 검증하는 것입니다.

핵심 구조: 공공데이터포털 상가정보 API → 카페/커피/음료 업종 필터 → 현재 위치 및 반경 계산 → 카카오맵 마커·원·목록 → GitHub → Render 공개 URL

2. 학습 목표와 완료 정의

  • 공공데이터 인증키와 Kakao JavaScript 키의 역할을 구분한다.
  • 기존 카카오맵 프로젝트의 실행·배포 상태를 점검한다.
  • 카페 후보 데이터 5건을 조회하고 필수 필드를 확인한다.
  • 현재 위치, 1km 원, 주변 카페 마커를 함께 표시한다.
  • 1km·3km·5km·전국 모드와 검색창을 연결한다.
  • 모바일 공개 URL에서 기능을 검증한다.
최종 완료 증거: 휴대폰에서 Render HTTPS 공개 URL을 열고 위치 권한을 허용했을 때, 내 위치 중심 1km 원·카페 마커·검색 결과 목록이 보이며 반경 버튼과 검색이 실제 결과를 바꾼다.

3. Gate 0 — 시작 전 필수 확인

이 Gate를 통과하지 못하면 실습 1로 진행하지 않는다.

  • Antigravity 2.0 설치 및 실행 가능
  • 8-1주차 프로젝트 폴더 접근 가능
  • Render 공개 URL에서 기존 카카오맵 표시
  • GitHub 저장소와 Render 연결 확인
  • Kakao Developers JavaScript 키 준비
  • 공공데이터포털 상가정보 API 승인 및 인증키 준비
금지: 인증키를 화면 캡처, 공개 GitHub, 프론트엔드 코드에 노출하지 않는다. Kakao Admin 키 대신 JavaScript 키를 사용한다.

4. 순차 실행 학습계획

1

기존 프로젝트 상태 확인

예상 20분진행 조건: Gate 0 통과
  • 8-1주차 카카오맵 프로젝트를 Antigravity 2.0에서 연다.
  • 카카오맵 로딩, Render 설정, GitHub 연결, 실행 방법, 패키지 상태를 점검시킨다.
  • 향후 API 연결에 적절한 구조를 제안받는다.
이 프로젝트는 8-1주차에서 만든 카카오맵 웹앱입니다. 제가 소스 코드를 직접 수정하지 않도록 현재 프로젝트 상태를 점검해 주세요. 카카오맵 로딩, Render 배포 설정, GitHub 연결, 실행 방법, 필요한 패키지, 공공 카페 데이터 연결 구조를 확인하고 필요한 설치와 실행은 직접 진행해 주세요. 제가 해야 할 클릭과 입력만 알려주세요.

성공 증거: 로컬 또는 미리보기에서 기존 카카오맵이 정상 표시되고, 에이전트가 실행 방법과 구조를 설명한다.

2

공공 카페 데이터 연결 및 5건 테스트

예상 40분보안 핵심
  • 상가정보 API에서 카페·커피·음료 업종만 필터링하도록 지시한다.
  • 인증키는 로컬 .env와 Render 환경변수로 분리한다.
  • 먼저 5건만 조회하여 상호명·업종명·주소·위도·경도를 확인한다.
소상공인시장진흥공단_상가(상권)정보_API를 연결해 주세요. 전국 상가 데이터 중 카페/커피/음료 업종만 검색합니다. 인증키는 화면과 공개 저장소에 노출하지 말고 로컬 환경변수와 Render 환경변수 방식으로 처리하세요. 먼저 5건만 조회해 상호명, 업종명, 주소, 위도, 경도가 있는지 실행 결과로 확인해 주세요.

성공 증거: 실제 응답에서 카페 후보 5건 내외와 필수 필드가 확인된다.

3

첫 화면을 현재 위치 기준 1km로 구성

예상 40분HTTPS 필요
  • 접속 시 위치 권한을 요청한다.
  • 허용 시 지도 중심 이동, 현재 위치 마커, 1km 원, 주변 카페 마커를 표시한다.
  • 거부 시 기본 위치와 쉬운 안내문을 제공한다.
첫 접속 시 브라우저 위치 권한을 요청하고, 허용하면 현재 위치를 지도 중심으로 설정해 현재 위치 마커와 1km 반경 원을 표시하세요. 1km 안의 카페만 기본 표시하세요. 거부 시 기본 위치와 안내 문구를 제공하고, HTTPS 공개 URL에서도 동작을 검증해 주세요.

성공 증거: 현재 위치 마커, 1km 원, 주변 카페 마커가 동시에 보인다.

4

1km · 3km · 5km · 전국 반경 칩 연결

예상 20분UI와 데이터 동기화
  • 기본 선택은 1km로 한다.
  • 칩 선택 시 원 크기·검색 반경·마커·목록을 함께 갱신한다.
  • 전국 모드는 반경 원을 숨기고 검색어가 있을 때만 제한된 결과를 보여준다.
1km, 3km, 5km, 전국 라벨식 칩을 추가하세요. 선택된 칩을 강조하고, 1/3/5km 선택 시 원 크기와 검색 결과, 마커, 목록을 함께 갱신하세요. 전국 모드는 검색어가 있을 때만 실행하고 결과 개수를 제한하세요. 모바일 터치에 적합하게 만드세요.

성공 증거: 칩을 누를 때 선택 표시뿐 아니라 실제 결과 수와 지도 표시 범위가 바뀐다.

5

카페 검색창과 결과 상호작용 구현

예상 15분검색 범위 연동
  • 상호명·주소·업종명에 검색어가 포함되면 결과를 표시한다.
  • 반경 모드에서는 선택 반경 내부만, 전국 모드에서는 제한된 전국 결과만 검색한다.
  • 목록 클릭 시 지도 이동 및 해당 마커 강조를 구현한다.
검색창과 검색 버튼을 추가하고 상호명, 주소, 업종명 기준으로 검색하세요. 현재 반경 모드와 검색을 연결하고, 결과 목록에 상호명, 주소, 업종명, 내 위치로부터의 거리를 표시하세요. 결과 클릭 시 지도 이동과 마커 강조를 구현하고, 결과가 없으면 반경 확대 안내를 보여주세요.

성공 증거: “스타벅스”, “커피”, “디저트”, “강남” 등의 검색어가 지도 마커와 목록을 함께 바꾼다.

6

초보자용 화면과 상태 메시지 정리

예상 15분사용성 검토
  • 제목, 현재 반경, 결과 개수, 검색창, 칩, 지도, 목록을 한눈에 배치한다.
  • 모바일 세로 화면에서 자연스러운 순서를 확인한다.
  • 로딩·빈 결과·오류 메시지를 쉬운 한국어로 제공한다.
전체 화면을 “내 주변 카페 찾기” 서비스로 정리하세요. 현재 검색 반경과 결과 개수를 표시하고, 검색창·반경 칩·지도·목록을 한눈에 배치하세요. 모바일에서는 검색 UI가 위, 지도와 목록이 아래에 오게 하며 로딩, 빈 결과, 오류 상태를 쉬운 문장으로 표시하세요.

성공 증거: 기능 확인용 화면이 아니라 처음 보는 사용자도 조작 가능한 서비스 화면처럼 보인다.

7

GitHub 반영, Render 재배포, 모바일 검증

예상 25분외부 변경 작업
  • 변경 파일 검토 후 적절한 커밋 메시지로 커밋·푸시한다.
  • Render 자동 배포 상태와 로그를 확인한다.
  • 배포 완료 후 데스크톱과 휴대폰에서 HTTPS 공개 URL을 검증한다.
변경 파일을 검토하고 적절한 커밋 메시지로 GitHub에 반영한 뒤 Render 자동 배포를 확인하세요. 배포 완료 후 공개 URL에서 위치 권한, 1km 기본 원, 반경 칩, 검색, 목록, 마커가 모두 작동하는지 점검하고 문제를 수정하세요.

성공 증거: 휴대폰 공개 URL에서 핵심 기능이 작동하며 인증키가 노출되지 않는다.

6. 검증 시나리오

  1. 첫 접속: 위치 권한 허용 → 현재 위치 마커·1km 원·카페 표시
  2. 권한 거부: 기본 위치와 안내문 표시, 앱이 멈추지 않음
  3. 반경 변경: 1km→3km→5km 순으로 결과와 원 크기 변화
  4. 전국 검색: 검색어 없이 과도한 호출 없음, 검색어 입력 후 제한된 결과
  5. 검색: 결과 목록과 지도 마커 동시 갱신
  6. 목록 클릭: 해당 카페로 지도 이동 및 마커 강조
  7. 보안: 브라우저 화면·저장소·로그에 인증키가 그대로 노출되지 않음
  8. 모바일: 터치 가능한 칩 크기, 읽기 쉬운 목록, HTTPS 위치 기능 확인
