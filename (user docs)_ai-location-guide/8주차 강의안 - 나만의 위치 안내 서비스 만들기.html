🤔 왜 이 수업을 하나요?

6주차는 파일 정리로 귀찮은 파일 관리를 줄였습니다. 7주차는 OCR로 사진 속 글자를 손으로 입력하는 일을 줄였습니다. 8주차는 내가 만든 앱을 인터넷에 공개해서, 다른 사람에게 위치를 설명하는 귀찮은 일을 줄입니다.

상황 1: 모임 장소 안내 “지하철 몇 번 출구에서 나와서…”라고 길게 설명하지 않고 링크 하나로 안내합니다.
상황 2: 내 가게 안내 지도, 영업시간, 전화번호, 주차 정보를 한 페이지에 보여줄 수 있습니다.
상황 3: 행사장 안내 강의장, 주차장, 식사 장소를 한 화면에 정리할 수 있습니다.
상황 4: 여행 장소 정리 맛집, 카페, 숙소, 관광지를 버튼으로 이동하는 지도 앱으로 만들 수 있습니다.

🧠 초보자에게 먼저 설명할 개념

1. localhost란?

localhost는 내 컴퓨터 안에서만 보이는 임시 주소입니다. 다른 사람은 내 localhost에 접속할 수 없습니다.

2. 공개 URL이란?

Cloud Run으로 배포하면 인터넷에서 접속할 수 있는 주소가 생깁니다. 예를 들면 아래와 같은 형태입니다. 

https://my-map-service-xxxxx.a.run.app

3. GitHub란?

GitHub는 내가 만든 앱 소스 파일을 클라우드에 기록하는 곳입니다. 쉽게 말하면 개발자용 구글 드라이브이자 작업 기록장입니다.

4. Cloud Run이란?

Cloud Run은 내가 만든 앱을 인터넷에서 실행해 주는 서비스입니다. 내 컴퓨터를 계속 켜두지 않아도, 인터넷 주소로 접속할 수 있게 해줍니다.

5. 카카오맵 API란?

카카오가 만든 지도 기능을 내 앱 안에서 사용할 수 있게 해주는 연결 통로입니다.

카카오맵 도메인 등록은 대표 도메인이 아니라 JavaScript SDK 도메인에 Cloud Run 공개 URL을 등록합니다.

이론 3: 왜 GitHub를 먼저 쓰는가?

이론 4: 왜 Cloud Run을 먼저 배포하는가?

카카오맵은 지도 사용을 허용할 도메인을 등록해야 합니다. 그래서 먼저 Cloud Run으로 공개 URL을 만들고, 그 URL을 카카오에 등록합니다.

1. 서울시청
2. 강남역
3. 홍대입구역

조건:
1. 지도 옆 또는 아래에 장소 버튼을 크게 보여 주세요.
2. 장소 버튼을 클릭하면 지도 중심이 해당 장소로 이동하게 해 주세요.
3. 마커도 해당 장소로 이동하게 해 주세요.
4. 현재 선택된 장소 이름을 화면에 크게 보여 주세요.
5. 각 장소에 간단한 설명을 넣어 주세요.
6. 모바일에서도 보기 좋게 만들어 주세요.

적용 후 다시 GitHub에 올리기

git add .
git commit -m "장소 목록 기능 추가"
git push
✅ 6단계 성공 기준
장소 버튼을 누를 때마다 지도가 이동하고 마커가 바뀌면 성공입니다.

실생활 확장 예시

가게 안내 앱 매장, 주차장, 가까운 지하철역 표시
강의장 안내 앱 교육장, 출입구, 식사 장소 표시
여행 지도 앱 맛집, 카페, 숙소, 관광지 표시
생활 편의 지도 병원, 약국, 주민센터, 복지관 표시

✨ 실습 7단계: AI에게 화면 정리 요청하기

마지막으로 앱을 보기 좋게 정리합니다. 기능이 먼저고 디자인은 나중입니다. 

현재 위치 안내 웹앱의 화면을 보기 좋게 정리해 주세요.

디자인 조건:
1. 제목은 크게 보여 주세요.
2. 지도는 화면 중앙에 크게 보여 주세요.
3. 장소 버튼은 큼직하게 만들어 주세요.
4. 현재 선택된 장소 설명을 카드 형태로 보여 주세요.
5. 모바일 화면에서도 보기 좋게 해 주세요.
6. 색상은 깔끔하고 초보자도 보기 쉽게 해 주세요.
7. 사용자가 무엇을 누르면 되는지 안내 문구를 넣어 주세요.

디자인 수정 후 GitHub에 올리기

git add .
git commit -m "위치 안내 앱 디자인 정리"
git push
✅ 최종 성공 기준
공개 URL에서 지도, 장소 버튼, 설명 카드가 보기 좋게 보이면 오늘 수업 성공입니다.

🧯 자주 막히는 문제와 해결법

문제 1. localhost에서는 되는데 Cloud Run에서 안 보여요

가능한 원인 Cloud Run 배포가 최신 코드로 다시 되지 않았을 수 있습니다.

해결 GitHub에 push가 되었는지 확인하고, Cloud Run에서 새 배포가 되었는지 확인합니다.

문제 2. 카카오맵이 하얗게만 보여요

가능한 원인 JavaScript 키가 틀렸거나 JavaScript SDK 도메인 등록이 빠졌을 수 있습니다.

해결 JavaScript 키와 Cloud Run URL 등록 상태를 다시 확인합니다.

문제 3. REST API 키를 넣었어요

가능한 원인 카카오 앱 키 종류를 헷갈린 것입니다.

해결 반드시 JavaScript 키를 복사해서 넣습니다.

문제 4. GitHub에 push가 안 돼요

가능한 원인 GitHub 로그인, 저장소 주소, 권한 문제가 있을 수 있습니다.

해결 저장소 주소가 https://github.com/아이디/저장소명.git 형태인지 확인합니다.

문제 5. Cloud Run에서 공개 접속이 안 돼요

가능한 원인 인증 필요 상태로 배포되었을 수 있습니다.

해결 실습용으로는 공개 접속 허용 또는 unauthenticated invocations 허용을 선택합니다.

문제 6. 장소 버튼을 눌러도 지도가 안 움직여요

가능한 원인 버튼 클릭 이벤트와 지도 이동 코드가 연결되지 않았을 수 있습니다.

해결 AI에게 “장소 버튼 클릭 시 지도 이동이 안 됩니다. 이벤트 연결을 다시 확인해서 수정해 주세요”라고 요청합니다.

🧑‍🏫 강사용 진행 팁

1. 오늘의 핵심은 카카오맵이 아닙니다

오늘의 핵심은 내 앱을 인터넷에 공개하고 외부 API를 연결하는 전체 흐름입니다. 카카오맵은 그 흐름을 눈에 보이게 해주는 좋은 예시입니다.

2. 초보자는 계정 로그인에서 많이 막힙니다

GitHub, Google Cloud, 카카오 계정 로그인이 모두 필요합니다. 수업 전 미리 로그인하도록 안내하면 실습 시간이 크게 줄어듭니다.

3. Cloud Run은 조교 지원이 필요합니다

Google Cloud 화면은 사용자마다 조금씩 다를 수 있습니다. 중급자 조교에게 Cloud Run 배포 단계 보조를 맡기는 것이 좋습니다.

4. API 키 오류는 코드보다 설정을 먼저 봅니다

지도가 안 뜨면 코드부터 고치지 말고, JavaScript 키와 JavaScript SDK 도메인 등록을 먼저 확인합니다.

5. 오늘의 금지사항

❌ Admin 키를 코드에 넣기 금지
❌ 비밀번호, 결제 정보, 개인정보를 GitHub에 올리기 금지
❌ 처음부터 길찾기, 검색, 현재 위치까지 한 번에 넣기 금지
❌ Cloud Run 공개 URL을 만들기 전에 카카오맵부터 붙이려고 하기 금지
