## Proposed Changes

### 1. 백엔드 연동 (`server.js`)
- **역할**: 프론트엔드가 공공데이터 API 키를 직접 호출하여 노출하지 않도록, 안전한 프록시 API 엔드포인트 `/api/cafes`를 구축합니다.
- **Node.js 내장 fetch 활용**: Node.js v24.18.0이 탑재되어 있으므로 추가 라이브러리(axios 등) 설치 없이 기본 내장 `fetch`로 데이터를 요청합니다.
- **하이브리드 업종 코드 대응**: 공공데이터 API의 최신 코드 체계인 `I212`(비알코올 음료점업)를 우선 조회하고, 결과가 없을 경우 기존 구 체계인 `Q12`(커피점/카페)로 자동 전환하여 조회(Fallback)하는 견고한 로직을 적용합니다.

### 2. 프론트엔드 UI & 연동 로직 (`public/index.html`)
- **1단계 (탐색 지역 지정)**: 기존 15개 시도 거점 드롭다운을 유지하면서, Geolocation API와 연동해 첫 접속 시 "내 현재 위치 기준 1km"가 기본 활성화되도록 합니다.
- **2단계 (주변 실시간 검색 및 반경 칩)**: 
  - `1km`, `3km`, `5km`, `전국` 형태의 프리미엄 칩 UI를 구성합니다.
  - 사용자가 선택한 반경에 따라 지도 위에 원(`kakao.maps.Circle`)을 그리고 범위(m)를 갱신합니다.
  - `전국` 모드 선택 시에는 지도 반경 원을 숨기고, 텍스트 검색어가 입력되었을 때만 제한된 수량의 결과를 전국에서 탐색하도록 제어하여 성능을 최적화합니다.
- **3단계 (백엔드 API 연동 및 검색)**:
  - 사용자가 지정한 중심점(lat, lng)과 반경(radius) 값을 `/api/cafes`로 요청하여 공공 카페 데이터를 가져옵니다.
  - 가져온 카페 정보(상호명, 업종명, 주소, 위도, 경도)를 지도 상에 마커로 렌더링하고, 왼쪽 결과 리스트에 표시합니다.
  - 목록 카드에 **내 현재 중심 위치로부터의 실제 거리**를 계산하여 직관적으로 표시합니다.
  - 로딩 상태 메시지("카페 데이터를 불러오는 중입니다..."), 데이터 없음 상태("검색 결과가 없습니다. 반경을 넓혀보세요.") 등의 예외 처리를 깔끔하게 구현합니다.

## Verification Plan

### Automated Tests
- 자가진단 검사 재실행:
  ```bash
  npx vibe-diagnosis run
  ```
  수정된 코드가 `project-diagnostic` 및 `ui-ux-diagnostic` 명세를 해치지 않고 통과하는지 확인합니다.

### Manual Verification
1. **로컬 서버 구동**: 
   ```bash
   node --env-file=.env server.js
   ```
2. **초기 진입 테스트**: `http://localhost:3000` 접속 후 브라우저 위치 권한을 허용했을 때, 내 위치 중심으로 1km 파란색 반경 원과 내 현재 위치 마커(별 모양 등)가 표시되는지 확인합니다.
3. **API 연동성 확인**: 내 현재 위치 반경 1km 내에 공공 카페 목록이 마커와 좌측 리스트에 렌더링되는지 확인합니다.
4. **반경 변경 테스트**: `3km`, `5km` 칩을 클릭했을 때 반경 원의 시각적 크기가 변경되고 리스트 데이터가 비동기 갱신되는지 확인합니다.
5. **텍스트 검색 연동**: 검색창에 "스타벅스" 등을 검색했을 때, 현재 선택된 반경(예: 1km) 내에서 해당 상호명을 가진 매장만 목록과 마커로 필터링되는지 검증합니다.
6. **전국 모드 검증**: `전국` 칩을 누르면 반경 원이 지워지고, 검색어를 입력했을 때 전국 범위에서 카페를 정상 조회해오는지 확인합니다.
