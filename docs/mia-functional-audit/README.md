# MIA 기능 감사 기록

동네나침반의 실제 사용자 기능, 의도적으로 주입한 실패, 발견된 결함,
구현 난제와 검증 결과를 보관하는 독립 섹션입니다.

## 기록 원칙

- 공개 서비스의 수정 전 상태와 로컬 수정 후 상태를 구분합니다.
- 코드 존재 여부가 아니라 실제 HTTP 요청과 브라우저 상호작용을 근거로 기록합니다.
- 성공뿐 아니라 빈 결과, 권한 거부, 외부 API 오류, 응답 경합을 함께 검사합니다.
- 인증키와 로컬 설정은 기록하거나 커밋하지 않습니다.
- Render 공개 URL에서 재검증하기 전에는 릴리스 완료로 표시하지 않습니다.
