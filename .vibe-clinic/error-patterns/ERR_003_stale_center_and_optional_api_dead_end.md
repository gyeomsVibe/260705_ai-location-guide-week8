# ERR-003: 지역 중심 불일치와 선택 API 막다른 화면

## 증상
- 지역 선택 UI는 바뀌지만 지도와 장소 검색이 즉시 같은 좌표로 갱신되지 않는다.
- 고정된 소수 지역만 선택할 수 있어 전국 탐색처럼 보이지만 실제 계약이 제한된다.
- 선택형 공공 API 키가 없으면 503과 안내 화면만 표시되어 핵심 혜택 탐색이 끝난다.
- DOM 요소 존재만 검사하는 진단은 이 결함을 정상으로 오판한다.

## 원인
- 위치 라벨 상태와 검색 좌표·재요청을 분리해 변경했다.
- 지역을 정적 좌표 배열로 모델링했다.
- 외부 API 실패를 사용자 가치가 유지되는 공식 경로 데이터로 대체하지 않았다.
- 진단이 실제 요청 파라미터와 사용자 상호작용을 검증하지 않았다.

## 교정
- 국내 범위 지오코딩 엔드포인트로 전국 지역을 검색한다.
- 지역 확정 시 지도 중심과 `/api/places` 좌표를 같은 트랜잭션으로 갱신한다.
- 혜택 키가 없을 때 정부24·K-Startup·소상공인24 공식 경로 카드를 200으로 제공한다.
- 빠른 연속 요청은 이전 요청을 중단하고 최신 컨트롤러만 상태를 갱신한다.

## 재발 방지 검사
- 지역 검색 후 첫 `/api/places`의 lat/lng가 지오코딩 결과와 일치해야 한다.
- 수도권·강원·전남·부산·제주 검색이 서로 다른 국내 좌표를 반환해야 한다.
- 혜택 네 프리셋이 키 없이도 각각 200과 공식 링크 카드를 반환해야 한다.
- 빈 배열·500·권한 거부·느린 이전 응답을 주입한 브라우저 및 단위 테스트를 유지한다.
