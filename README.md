# 📍 나만의 위치 안내 & 주변 검색 서비스 (Week 8)

전국 17개 시·도의 주요 거점(총 **89개 상세 지점**)과 **내 현재 위치(GPS)** 를 기준으로, 그 주변의 생활 편의 시설과 모임 장소를 **카카오맵으로 실시간 검색**해 주는 웹 서비스입니다.
단순한 정적 위치 안내를 넘어, 카카오맵 Services API를 연동한 동적 검색과 반응형 UI를 제공합니다.

🔗 **배포 주소**: [https://ai-location-guide-week8.onrender.com/](https://ai-location-guide-week8.onrender.com/)

---

## ✨ 주요 기능

### 1. 2단계 지역 선택 (권역 → 상세 지점)
- **1단계 권역**: 내 현재 위치 + 전국 14개 시·도(서울🏙️·인천✈️·부산🌊·대전🔬 등 지역별 **상징 아이콘**으로 구분).
- **2단계 상세 지점**: 선택한 권역의 대표 랜드마크를 **지역당 4~9개**(전국 총 89개) 제공합니다.
  - 예) 서울: 서울시청·강남역·홍대·여의도·잠실·명동·광화문·서울역·성수동
  - 예) 인천: 계양구청·계산역·부평구청·부평역·송도센트럴파크·인천시청·인천공항·월미도·계양산
- 공간을 아끼는 Shadcn UI 스타일 드롭다운(포커스 링, 쉐브론 화살표)을 바닐라 CSS로 구현했습니다.

### 2. 주변 실시간 검색 (반경 2km)
- 검색창에 원하는 장소·키워드(다이소, 도서관, 소방서 등)를 직접 입력해 **선택 지점 2km 반경**을 실시간 조회합니다.
- 자주 찾는 **추천 카테고리 칩 9종**으로 원클릭 검색:
  - 🏛️ 생활·행정: 주민센터 · 병원 · 약국 · 은행 · 마트
  - 🥕 모임·나들이: 맛집 · 카페 · 공원 · 등산

### 3. 인터랙티브 지도 + 결과 목록 연동
- 검색 결과를 **다중 마커**로 지도에 표시하고, 좌측 목록 카드에 상호명·도로명 주소·중심점 거리(m/km)·카카오맵 상세 링크를 제공합니다.
- 목록 카드나 마커를 누르면 해당 위치로 이동하며 **커스텀 오버레이(툴팁)** 가 열립니다.
- 지역·검색어를 바꾸면 기존 마커·오버레이·목록이 깔끔하게 초기화됩니다.

### 4. 내 위치 기반 서비스 (LBS)
- 접속 시 브라우저 GPS로 현재 위치를 자동 감지하여 중심으로 설정하고, 드롭다운의 "📍 내 현재 위치 기준"으로 언제든 갱신할 수 있습니다.
- 내 위치는 일반 지점과 구별되도록 **별 모양 마커**로 표시합니다.

## 🖥️ 화면 설계 및 UX

- **분할 레이아웃**: 데스크톱에서는 좌측 480px 조작·결과 패널 + 우측 지도 영역의 2분할 구조.
- **모바일 반응형**: 좁은 화면에서는 상단 지도 + 하단 스크롤 조작 영역으로 재배치되어 한 손 조작에 최적화됩니다.
- **깔끔한 컬러**: 인디고 기반의 밝고 시인성 높은 톤으로 초보자도 보기 쉽게 구성했습니다.

## 🚀 로컬 실행 방법

### 1. 사전 준비
- [Node.js](https://nodejs.org) 설치 (LTS 권장)

### 2. 패키지 설치
```bash
npm install
```

### 3. 서버 실행
```bash
npm start
```
- 기본 **3000번 포트**에서 실행됩니다. (`server.js`는 `process.env.PORT`가 있으면 그 값을, 없으면 3000을 사용 → Render 등 배포 환경에서는 플랫폼 지정 포트로 자동 동작)

### 4. 브라우저 접속
```text
http://localhost:3000
```

> ⚠️ 로컬에서 지도가 안 보이면, 카카오 개발자 콘솔 → 플랫폼 키 → **JavaScript SDK 도메인**에 `http://localhost:3000` 을 등록해야 합니다. (배포 도메인은 이미 등록되어 있어 라이브 사이트는 정상 동작합니다.)

## 🛡️ 운영 및 보안

- **동적 포트 바인딩**: `process.env.PORT` 우선 참조로 로컬/배포 포트 불일치 문제를 방지합니다.
- **버전 헤더 차단**: `x-powered-by` 헤더를 비활성화해 Express 버전 노출을 막았습니다.
- **HTML 캐시 방지**: `index.html`에 `Cache-Control: no-store` 계열 헤더를 적용해, 배포 직후 이전(캐시) 버전이 겹쳐 보이는 문제를 방지합니다.
