# 📍 나만의 위치 안내 & 주변 검색 서비스 (Week 8)

인천 권역(계양구, 부평구)과 서울 권역(시청, 강남, 홍대)을 한눈에 아우르며, 사용자가 지정한 지역 주변의 편의 시설 및 모임 장소를 실시간으로 검색해주는 웹 서비스입니다.
기존의 단순 정적 장소 안내를 탈피하여, 카카오맵 API(Kakao Maps Services API)를 연동한 동적 검색과 고품질의 반응형 UI를 제공합니다.

🔗 **배포 서비스 주소**: [https://ai-location-guide-week8.onrender.com/](https://ai-location-guide-week8.onrender.com/)

---

## ✨ 주요 기능

1. **권역별 중심지 빠른 탐색 (Tab & Grid)**
   - **인천 권역**: 계양구청, 계산역, 부평구청, 부평역
   - **서울 권역**: 서울시청, 강남역, 홍대입구역
   - 지역 탭과 세부 버튼 클릭 시 지도가 해당 위치로 부드럽게 화면 이동(`panTo`)하며 정보 창이 표출됩니다.

2. **주변 실시간 카테고리 검색 (반경 1.5km)**
   - **🏛️ 생활·행정**: 주민센터, 병원, 약국, 은행, 마트
   - **🥕 모임·나들이**: 맛집, 카페, 산책(공원), 등산
   - 중심지 기준 1.5km 반경 내에 존재하는 장소들을 실시간으로 카카오 지도 데이터베이스에서 필터링 조회합니다.

3. **인터랙티브 지도 및 상세 목록 연동**
   - 다중 마커(Multi-Marker)를 통해 지도 위에 검색 장소들을 도식화합니다.
   - 검색 결과 카드 리스트에는 상호명, 도로명 주소, 중심점으로부터의 거리(m/km 단위 변환), 카카오맵 상세 정보 페이지 아웃링크 버튼이 제공됩니다.
   - 결과 리스트 카드를 누르면 지도 상의 해당 마커 위치로 포커싱되며 툴팁(커스텀 오버레이)이 팝업됩니다.
   - 지역이나 카테고리를 새로 변경할 시, 기존에 떠 있던 마커와 리스트 잔여물이 깔끔하게 소거(Clear)되도록 설계되었습니다.

## 🖥️ 화면 설계 및 프리미엄 UX

- **분할 화면 레이아웃 (Split Layout)**: 가로 영역이 충분한 데스크톱 브라우저에서는 좌측 420px 영역에 조작 패널과 결과 스크롤 리스트가 고정되고, 우측 100% 영역에 지도가 차오르는 대화형 레이아웃을 제공합니다.
- **모바일 반응형 설계**: 화면 가로 폭이 좁은 모바일에서는 최상단에 지도가 고정 배치되고, 하단에 스크롤 형식의 제어 영역이 이어져 한 손 조작 편의성을 극대화합니다.
- **리치 에스테틱 컬러**: 인디고 테마를 기본으로 하되 생활 행정은 민트, 모임 나들이는 오렌지색 악센트를 주어 시인성이 높고 부드러운 글래스모피즘 분위기를 적용했습니다.

## 🛠️ 기술 스택 및 오픈 API

- **Frontend**: Vanilla HTML5, CSS3 Grid/Flex, Javascript (ES6+)
- **Backend**: Node.js & Express Framework
- **API**: Kakao Maps SDK (Maps, Services Library)
- **Deployment**: Render.com & GitHub Auto Deployment Pipeline

## 🚀 실행 방법 (로컬 환경)

### 1. 사전 준비

- [Node.js](https://nodejs.org) 설치 (권장 버전: LTS)

### 2. 패키지 설치

프로젝트 루트 폴더 경로의 터미널에서 다음 명령어를 입력합니다:

```bash
npm install
```

### 3. 로컬 서버 시작

```bash
npm start
```

- 기본적으로 **3001번 포트**에서 가동됩니다. (로컬 환경의 3000번 포트 충돌 문제를 피하기 위해 3001번 포트에서 가동되도록 수정되었습니다.)

### 4. 웹 브라우저 접속

서버 가동 후 브라우저를 통해 아래 주소로 접속합니다:

```text
http://localhost:3001
```

## 🛡️ 운영 및 보안 개선 사항

- **포트 바인딩 최적화**: 로컬 개발과 서버 배포 환경의 포트 불일치 문제를 해결하기 위해 `process.env.PORT`를 우선 참조하도록 설계하였습니다.
- **버전 노출 차단**: Express 프레임워크의 취약점이 노출되지 않도록 `x-powered-by` 헤더를 백엔드 서버에서 비활성화 처리하였습니다.
